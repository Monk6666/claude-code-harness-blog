# Claude Code Harness Blog — 代碼架構側寫與分析

日期：2026-04-02

## 1) 專案定位

### 可確認事實
- 此 repo 是一個以 **Astro + Starlight** 建置的技術文件網站，主題聚焦「Claude Code Harness Engineering」。
- 首頁採用 `splash` 模板，提供 14 個章節導讀卡片與架構圖元件。
- 網站的實際核心內容是 `src/content/docs/` 下的 MDX 章節。

### 推測
- 這個 repo 的「產品」不是執行 Claude Code 本體，而是「對其架構進行知識化整理」的出版型站點。

### 系統邊界
- **邊界內**：文件內容（MDX）、Starlight 網站組態、少量客製化元件/樣式。
- **邊界外**：章節中反覆引用的 `src/query.ts`、`src/Tool.ts` 等「Claude Code 原始碼」並不在本 repo 內。

## 2) 技術棧與依賴

### 可確認事實
- 主要語言：TypeScript（設定檔）、MDX（內容）、Astro component（`.astro`）。
- 框架與依賴：`astro`, `@astrojs/starlight`, `@astrojs/react`, `astro-mermaid`, `sharp`。
- 執行腳本：`astro dev/build/preview`。

### 架構風格判讀
- 明顯是 **content-driven static docs architecture**：
  - Starlight 負責文件資訊架構（sidebar autogenerate）。
  - MDX 內容為主，元件僅補充展示（架構圖、社群圖示）。

## 3) 目錄與模組責任

### 主要目錄
- `src/content/docs/`：主文件內容（首頁 + 14 章）。
- `src/components/`：客製 UI 元件（`ArchDiagram`, `SocialIcons`）。
- `src/styles/`：全域樣式覆寫。
- `public/`：靜態資產。
- `docs/plans`, `docs/drafts`, `docs/autoresearch-blog`：協作與產研過程文檔。

### 核心模組 vs 支援模組
- **核心模組**：
  1. `astro.config.mjs`（站點整體裝配）
  2. `src/content.config.ts`（content collection schema）
  3. `src/content/docs/**`（實際產品內容）
- **支援模組**：
  - `src/components/*`、`src/styles/custom.css`（視覺/互動增強）
  - `README.md`（目前內容與實際技術棧不一致）

## 4) 入口與執行流程

### 主入口
1. `npm run dev` / `npm run build` 觸發 Astro CLI。
2. Astro 讀取 `astro.config.mjs`，載入 `mermaid`, `starlight`, `react` integrations。
3. Starlight 依 `src/content.config.ts` 識別 docs collection 並建構路由/側欄。
4. 首頁載入 `src/content/docs/index.mdx`，其中引入 `ArchDiagram.astro`。
5. 頁首社群區載入 `SocialIcons.astro`，於 client 端呼叫 GitHub API 抓 star 數。

### 核心執行鏈路（簡化）
- Build-time: `Astro config -> Starlight content discovery -> MDX compile -> static output`
- Runtime (browser): `SocialIcons script -> fetch GitHub repo metadata -> update DOM`

### 事件流/回呼補充
- 明確的 runtime callback 僅見於 `SocialIcons` 的 `loadStars()` 非同步流程。
- 章節內提及的 query loop/FSM/event bus 為「被分析對象（Claude Code）」敘事，不是本 repo 的執行機制。

## 5) 核心抽象與設計模式

### 可確認事實
- `docsSchema()`：以 schema 驅動內容結構。
- `sidebar.autogenerate`：以目錄結構驅動導覽生成。
- component injection：透過 `components.SocialIcons` 覆寫 Starlight 預設元件。

### 推測
- 本 repo 偏向「組態即架構（Configuration-as-Architecture）」：
  - 以 config + content metadata 控制整站行為，非重業務邏輯程式碼。

## 6) 代碼架構側寫

### 高穩定（骨架）
- `astro.config.mjs`
- `src/content.config.ts`
- `src/content/docs/chapters` 的編排規則（order + sidebar autogenerate）

### 高變動（邊界層）
- `src/content/docs/chapters/*.mdx`：內容會持續迭代。
- `docs/plans`, `docs/drafts`：明顯屬於工作流中間產物。

### 擴充點
1. 新增章節：在 `src/content/docs/chapters/` 新增 MDX + `sidebar.order`。
2. 新增互動視覺：在 MDX 中 import `.astro` / React component。
3. 新增語系：擴充 `astro.config.mjs` 的 `locales`。
4. 新增社群/觀測：延伸 `SocialIcons` 或自定 header/footer component。

### 耦合點
- 首頁與章節 slug 命名耦合（LinkCard `href` 需與實際路由同步）。
- `SocialIcons` 對 GitHub API 成功率與 rate-limit 有耦合。
- README 與實際框架不一致，形成 onboarding 認知耦合/落差。

### 潛在技術債
1. `README.md` 仍是 Next.js 範本，與實際 Astro 專案不符。
2. 章節大量引用 repo 內不存在的程式碼路徑（如 `src/query.ts`），對讀者可追溯性造成斷裂。
3. 缺少內容品質自動檢查（例如 broken links、引用檔案存在性驗證）。

## 7) 閱讀路線（首次接觸建議 10 檔）

1. `package.json`：確認專案類型與指令。
2. `astro.config.mjs`：掌握組態骨架與 integrations。
3. `src/content.config.ts`：理解內容資料模型。
4. `src/content/docs/index.mdx`：掌握首頁資訊架構與導航策略。
5. `src/components/ArchDiagram.astro`：理解「概念模型」如何可視化。
6. `src/components/SocialIcons.astro`：唯一明顯 runtime 行為（API fetch）。
7. `src/styles/custom.css`：主題品牌化與版型覆寫。
8. `src/content/docs/chapters/01-introduction.mdx`：內容體系總覽。
9. `src/content/docs/chapters/07-coordinator-concurrency.mdx`：內容深度代表章。
10. `README.md`：辨識文檔與實況差異（重要風險訊號）。

## 8) 總評

### 架構成熟度
- **網站骨架成熟度：中高**（Starlight 標準化 + 清楚 config）。
- **知識產品成熟度：中**（內容深但可追溯性尚可加強）。

### 維護性
- 整體維護成本低（多為內容維護）。
- 主要風險在文檔一致性與引用真實性，不在程式複雜度。

### 擴充性
- 對新章節、主題、元件擴充友善；
- 若要進一步產品化（全文搜尋優化、觀測、版本切換），仍有良好延展空間。

### 測試性
- 程式邏輯面測試需求低；
- 內容品質更應靠 link-check、schema lint、build gate 保障。

### 二次開發難度
- **低到中**：前提是熟悉 Astro/Starlight 的 content pipeline。
- 真正難點在「內容工程流程」而非「執行時系統工程」。

## 簡化文字流程圖

```text
Developer writes MDX
    -> Starlight docs collection parses frontmatter/schema
    -> Sidebar/routes autogenerated from directory/order
    -> Astro build renders static docs pages
    -> Browser loads page
    -> SocialIcons fetches GitHub stars (optional runtime enhancement)
```

## 簡化模組關係圖

```text
package.json scripts
   -> astro.config.mjs
      -> Starlight integration
         -> src/content.config.ts (docs schema)
         -> src/content/docs/** (MDX content)
         -> src/components/SocialIcons.astro (header social override)
      -> mermaid integration (diagram rendering in MDX)
      -> react integration (optional interactive islands)

src/content/docs/index.mdx
   -> imports ArchDiagram.astro
   -> links all chapter routes
```
